21 PURPOSE:
22 ----------------------------------------------------------
23 The purpose of this package is to leverage the Standards
24 Of Fundamental Astronomy (SOFA) package in order to compute
25 several astrophysical quantities regarding the positions of
26 objects.
27 
28 Here is a list of purposes I wish this code to serve:
29 * Basic coordinate conversion routines (see CECoordinates, fully implemented)
30  - __CIRS__ (Earth centric RA,Dec)
31  - __ICRS__ (Solarsystem barycentric RA, Dec)
32  - __Galactic__ (Long, Lat)
33  - __Observed__ (Azimuth, Zenith angle) (Note: Zenith angle = 90&deg; - Altitude)
34 * Star & Planet ephemeris (currently not implemented, 3rd priority)
35  - Star & Planet positions for a given observer at a given time
36 * Basic time conversion routines (currently not implemented, 2nd priority)
37  - UTC
38  - Local time
39  - Greenwich apparent sidereal time
40  - Local apparent sidereal time
41 * Date conversion routines (see CEDate, fully implemented)
42  - __Julian Date__
43  - __Modified Julian Date__
44  - __Gregorian Calendar__ (year, month, day)
45 
46 More feature will be implemented as time permits. If there
47 is any feature that you would like to see implemented feel
48 free to contact the author or submit an issue and I'll
49 look into it.
50 
51 The following is a list of currently fully implemented, compiled
52 executables which can be run from the command line:
53 * Date conversion routines
54  - __cal2jd__: Gregorian calendar to Julian date
55  - __cal2mjd__: Gregorian calendar to modified Julian date
56  - __jd2cal__: Julian date to Gregorian calendar date
57  - __jd2mjd__: Julian date to modified Julian date
58  - __mjd2cal__: Modified Julian date to Gregorian calendar date
59  - __mjd2jd__: Modified Julian date to Julian date
60 * Coordinate conversion routines (all angles are expected in degrees)
61  - __cirs2obs__: CIRS to Observed coordinates
62  - __cirs2gal__: CIRS to Galactic coordinates
63  - __gal2cirs__: Galactic to CIRS coordinates
64  - __gal2obs__: Galactic to Observed coordinates

66 Downloading the code:
67 ----------------------------------------------------------
68 To obtain the code, it should be as simple as cloning the

70 ```bash
71 git clone https://github.com/Jvinniec/CppEphem.git CppEphem

74 Building the code:
75 ----------------------------------------------------------
76 As long as the user has the most up to date version of autotools,
77 you should be able to build the software very easily using the
78 standard "./configure -> make -> make install" method. Here is
79 a bit more detail.
80 
81 First, download the repository as described above in "Downloading
82 the code". Second, make sure that the "configure" file exists in
83 the top directory. If not, then do:
84 
85 ```bash
86 . autogen.sh
87 ```
88 
89 Third, configure the software (note the "prefix" option is optional):
90 
91 ```bash
92 ./configure [--prefix=/your/install/directory]
93 ```
94 
95 Fourth, build the code:
96 
97 ```bash
98 make
99 ```
100 
101 Finally, to install the code in an accesible manner (i.e. so that
102 your PATH and (DY)LD_LIBRARY_PATH environment variables know where
103 the executables and libraries are) type:
104 
105 ```bash
106 make install
107 ```
108 
109 And that should do it.
110 
111 Uninstalling the code:
112 ----------------------------------------------------------
113 To uninstall the code, it is advised to first run
114 
115 ```bash
116 make uninstall
117 ```
118 
119 in the top directory in order to remove the executables from your
120 base install directory. Then you can delete the downloaded git repository.
